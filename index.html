Featured project

Allianz chatbot improvements

A reliability-focused RAG project for internal insurance documents, including multimodal inputs. The work combined retrieval improvements, evaluation design, and production-minded system building.

  • Extended the pipeline with GraphRAG / knowledge-graph context selection to improve retrieval relevance.
  • Built automated evaluation combining reward modeling, human review, and LLM-as-a-judge workflows.
  • Worked in a modular Python / PyTorch setup with REST services, CI/CD, Docker, Kubernetes, and Azure deployment constraints.
View repository

What mattered here was not just adding another RAG trick. The project was about making answers easier to trust: improving context selection, reducing brittle behavior, and putting evaluation at the center of iteration.

Research Deep RL PyTorch

Dormant neurons in deep RL

Replication and extension work around dormant neuron behavior across PPO, SAC, and DQN, with an emphasis on instrumentation and reproducibility.

  • Tracked τ-dormancy and gradient dormancy during training with lightweight custom tooling.
  • Implemented ReDo-style neuron recycling with targeted reinitialization, optimizer-state reset, and trigger ablations.
  • Measured effects on returns, stability, and feature collapse under sparse-reward and replay-heavy regimes.

Systems Agentic RAG MCP

MCP server for agentic RAG

An MCP-compliant server built to expose tools safely to LLM agents, with a strong focus on observability and reproducible evaluation.

  • Integrated with LangChain and LangGraph for tool use in agent workflows.
  • Added JSON-RPC and WebSocket observability to inspect tool-selection behavior and agent traces.
  • Benchmarked latency, throughput, and tool-call reliability under simulated workloads.

Current direction

AI safety and interpretability

I am especially interested in methods that make model behavior more legible: identifying risky features, testing interventions, and measuring the safety–utility trade-off instead of talking about it abstractly.

In progress SAEs Open LLMs

Causal AI safety with sparse autoencoders

Ongoing work on using pretrained SAE suites to surface interpretable latent features associated with jailbreak or harmful intent in open, quantized language models.

  • Extract candidate latent features and cluster them from top-activating prompts.
  • Run suppression / steering interventions at inference time.
  • Measure safety gains against utility loss on a compact HarmBench-style evaluation set.
